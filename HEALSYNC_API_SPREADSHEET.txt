DETAILED API DOCUMENTATION
==========================

PATIENT AUTHENTICATION APIS
============================

1. Patient Login
   - Method: POST
   - URL: https://healsync-backend-d788.onrender.com/v1/healsync/patient/login
   - Headers: Content-Type: application/json
   - Request Body:
     {
       "email": "patient@example.com",
       "password": "password123"
     }
   - Response (Success):
     {
       "patientId": 1,
       "patientName": "John Doe",
       "patientAge": 30,
       "gender": "Male",
       "mobileNo": "1234567890",
       "email": "patient@example.com",
       "treatmentPlanIds": [1, 2]
     }
   - Response (Error): 401 "Invalid email or password"

PATIENT MANAGEMENT APIS
=======================

2. Add Patient (Registration)
   - Method: POST
   - URL: https://healsync-backend-d788.onrender.com/v1/healsync/patient/add

   - Request Body:
     {
       "patientName": "John Doe",
       "patientAge": 30,
       "gender": "Male",
       "mobileNo": "1234567890",
       "email": "patient@example.com",
       "password": "password123"
     }

TESTING NOTES
=============

1. Most APIs require JSON content type header
2. Path parameters should be replaced with actual IDs
3. Query parameters are appended to URL with ? and &
4. Date formats should be in ISO format (YYYY-MM-DDTHH:mm:ss)
5. All APIs return JSON responses
6. Error responses typically include HTTP status codes and error messages

HTTP STATUS CODES USED
======================
- 200 OK: Successful operation
- 400 Bad Request: Invalid request data
- 401 Unauthorized: Authentication failed
- 404 Not Found: Resource not found
- 500 Internal Server Error: Server error
